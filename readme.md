# 🛒 의류 재고 관리 시스템 데이터베이스 설계 프로젝트 (Clothes Inventory Management System DB Design)

## 1. 프로젝트 개요

### 1.1. 프로젝트 목표
본 프로젝트는 기존 의류 매장의 **수동적인 재고 관리 및 잦은 재고 불일치 문제**를 해결하기 위해, **데이터 무결성**과 **실시간 추적**을 최우선 목표로 하는 관계형 데이터베이스(RDB)를 설계하고 구현하는 것을 목표로 합니다.

### 1.2. 주요 기능
* **재고 정확성 확보:** 입고, 출고, 재고 조정 등 모든 재고 변동 트랜잭션을 기록하여 데이터의 불일치(Anomaly)를 방지합니다.
* **실시간 재고 추적:** 상품을 **옵션(색상/사이즈)** 및 **창고별**로 세분화하여 실시간 재고 현황 및 안전 재고 미달 여부를 즉시 파악합니다.
* **업무 효율화:** 재고 경고 리스트 및 관리자별 업무 이력 조회를 통해 운영 효율성을 높입니다.

## 3. 데이터베이스 설계 및 구조

### 3.1. 설계 특징
* **정규화:** 모든 테이블은 데이터 중복 및 이상 현상을 제거하기 위해 **제 3 정규형(3NF)**을 만족하도록 설계되었습니다.
* **약한 엔터티:** 상품의 재고 최소 단위인 `PRODUCT_OPTION`은 `PRODUCT`의 **약한 엔터티**로 설계되었습니다. (복합키: `{ProductID, SeqNum}`)
* **물리적 최적화:** `CurrentStock < SafetyStock` 조건이나 조인(`FK`)이 빈번한 컬럼에 **인덱스**를 설정하여 조회 성능을 극대화했습니다.

### 3.2. 핵심 테이블 스키마 요약

| 테이블 명 | 역할 | 기본 키 (PK) | 주요 외래 키 (FK) |
| :---: | :---: | :---: | :---: |
| **PRODUCT** | 상품 기본 정보 | `상품ID` | `브랜드ID`, `카테고리ID` |
| **PRODUCT_OPTION** | 색상/사이즈 옵션 정보 | `상품ID`, `순번` | `상품ID` |
| **WAREHOUSE** | 재고 보관 장소 | `창고ID` | `매장ID` |
| **INVENTORY** | **실시간 재고 현황** | `상품ID`, `순번`, `창고ID` | `상품ID/순번`, `창고ID` |
| **OUTBOUND** | 고객 주문 트랜잭션 | `출고ID` | `고객ID`, `관리자ID` |
| **ADJUSTMENT_HISTORY** | 재고 조정 기록 | `재고조ID` | `상품ID/순번`, `창고ID`, `관리자ID` |

## 4. 주요 구현 기능 (SQL DML 검증)

구현된 DML 쿼리를 통해 재고 관리의 핵심 기능을 검증했습니다.

### 4.1. 재고 분석 및 경고 기능
| No. | 기능 설명 | SQL 구현 유형 |
| :---: | :--- | :--- |
| **24** | **재고 부족 경고 리스트 조회** (안전 재고 미달 상품 식별) | `SELECT`, `JOIN`, `WHERE` (조건 필터링) |
| **10** | 재고가 0인 상품 옵션 목록 조회 | `SELECT`, `WHERE` |
| **19** | 창고별 총 재고량 순위 조회 | `GROUP BY`, `SUM`, `ORDER BY` |

### 4.2. 이력 및 트랜잭션 추적
| No. | 기능 설명 | SQL 구현 유형 |
| :---: | :--- | :--- |
| **18** | 최대 입고 트랜잭션을 처리한 관리자 정보 조회 | `MAX`, `SUBQUERY`, `JOIN` |
| **12** | 특정 입고를 처리한 관리자의 이름 및 권한 조회 | `JOIN` (3개 테이블) |
| **23** | 특정 상품을 구매한 고객이 이용한 관리자 추적 | `EXISTS`, `SUBQUERY` |
